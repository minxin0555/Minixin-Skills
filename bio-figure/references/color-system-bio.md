# Color system — journal palettes + algorithmic same-hue sequential

## 1. Decision rules (AI decides automatically; user can override)

| Data semantic | Palette type | Strategy |
|---|---|---|
| Single series / single condition | Single hue | Journal first colour (§2) |
| Two-class comparison | Two colours | Journal pair, default blue-orange (colorblind-safe) (§3) |
| Unordered categories ≥3 | Qualitative | **Default Okabe-Ito (colorblind-safe).** Switch to NPG / AAAS only if user wants journal look — **warn it is not colorblind-safe.** |
| Continuous / ordered | Sequential same-hue | Algorithm: fix hue, chroma↓ / luminance↑ (`palette.py sequential`) |
| Diverging with neutral centre | Diverging | Blue ↔ faint grey ↔ red (or purple ↔ green). On a white page, use a faint grey centre, not pure white. |

## 2. Journal first colours

| Journal | Main colours (hex) |
|---|---|
| NPG (Nature) | `#E64B35` red / `#4DBBD5` blue |

| Lancet | `#00468B` blue / `#ED0000` red |
| Colorblind-safe | `#0072B2` blue / `#E69F00` orange (Okabe-Ito) |

## 3. Two-class pairs

| Scheme | A | B | Use |
|---|---|---|---|
| Blue-orange (default, CB-safe) | `#0072B2` | `#E69F00` | case/control, binary |
| Red-blue (journal high frequency) | `#E64B35` | `#4DBBD5` | up / down regulated |
| Deep-blue / red (clinical) | `#00468B` | `#ED0000` | medical contrasts |
| Teal / magenta (CB-safe alt) | `#009E73` | `#CC79A7` | avoid red-green clash |

## 5. Sequential same-hue and diverging

Generated by `scripts/palette.py`, no third-party deps:

- `palette.py sequential --hue <hex-or-hue-deg> -n 7` — fix hue, chroma descends, luminance ascends.
- `palette.py diverging --low <hex> --high <hex> -n 9` — symmetric ramp through a faint-grey centre.
- Suggested two-side hues for diverging: blue `#0072B2` ↔ red `#D55E00`; or teal `#009E73` ↔ magenta `#CC79A7`.

## 6. CLI of `palette.py`

```
palette.py sequential  --hue <hex|deg> -n 7
palette.py diverging   --low <hex> --high <hex> -n 9
palette.py two-color   [--seed <hex>]               # no seed → defaults to blue/orange
palette.py qualitative -n 8 [--journal npg|aaas|nejm|lancet|jco|okabe-ito|tableau10]
palette.py check       <hex...>                     # protan / deutan / tritan simulation + WCAG contrast
palette.py swatch      <hex...> -o card.png         # render a swatch PNG (uses matplotlib)
```

Colour-space conversions and colorblind simulations are written from scratch in pure standard library. Only `swatch` reaches for matplotlib to render the PNG.

## 7. Swapping palettes / showing options

When the user says "give me a few options" or "change the colour":

1. Generate 2–4 candidates by §1 rules.
2. Render each as a **swatch PNG** (`palette.py swatch …`) annotated with hex + usage.
3. Show them, ask the user to pick.
4. Replot with the chosen set.

## 8. Colorblind safety

Always run `palette.py check` on any non-Okabe-Ito qualitative palette and:

- If the protan / deutan / tritan simulation collapses two key categories, **warn**.
- If the user insists on a non-safe journal palette, keep the warning in the report (§`export-bio.md` QA checklist).
